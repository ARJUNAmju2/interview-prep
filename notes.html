Key Concepts Quick Reference

OOP 4 Pillars

  • Encapsulation - Hide data, use get/set. Example: Store Module stock is private
  • Inheritance - Child gets parent features. Example: BaseController in Audit Module
  • Polymorphism - Same name, different behavior. Example: Different ticket types in Helpdesk
  • Abstraction - Show only needed. Example: ITicketService interface

SOLID Principles

  • S - Single Responsibility: One class = one job
  • O - Open/Closed: Extend without modifying
  • L - Liskov: Child can replace parent
  • I - Interface Segregation: Small focused interfaces
  • D - Dependency Inversion: Depend on interfaces, not concrete classes

SQL Joins

  • INNER JOIN - Only matching rows from both tables
  • LEFT JOIN - All from left + matching from right (NULL if no match)
  • RIGHT JOIN - All from right + matching from left
  • FULL JOIN - All from both (NULL where no match)
  • CROSS JOIN - Every row combined with every row (cartesian product)

DELETE vs TRUNCATE vs DROP

  • DELETE - Remove specific rows, can rollback, slow
  • TRUNCATE - Remove ALL rows, cannot rollback, fast
  • DROP - Remove entire table permanently

MVC .NET Core Key Concepts

  • Controller - Handles requests, returns responses
  • Model - Data structure (classes)
  • View - UI (Razor pages)
  • DI - AddScoped, AddSingleton, AddTransient
  • Middleware - Request pipeline (auth, logging, error handling)
  • EF Core - ORM for database operations

C# Abstraction

Abstract Classes and Methods

Data abstraction is the process of hiding certain details and showing only essential information to the user. Abstraction can be achieved with either abstract classes or interfaces.

  • Abstract class: A restricted class that cannot be used to create objects. It must be inherited from another class.
  • Abstract method: Can only be used in an abstract class, and it does not have a body. The body is provided by the derived class.
  • An abstract class can have both abstract and regular methods.

Example - Abstract Class

abstract class Animal
{
  public abstract void animalSound(); // No body - child MUST implement
  public void sleep()
  {
    Console.WriteLine("Zzz");
  }
}

You cannot create an object of abstract class:

Animal myObj = new Animal(); // ERROR! Cannot create instance of abstract class

How to Use - Inherit and Override

Use : to inherit and override keyword to provide the body:

// Abstract class
abstract class Animal
{
  public abstract void animalSound(); // abstract method
  public void sleep()
  {
    Console.WriteLine("Zzz");
  }
}

// Derived class (inherit from Animal)
class Pig : Animal
{
  public override void animalSound()
  {
    Console.WriteLine("The pig says: wee wee");
  }
}

// Usage
class Program
{
  static void Main(string[] args)
  {
    Pig myPig = new Pig();
    myPig.animalSound(); // Output: The pig says: wee wee
    myPig.sleep();       // Output: Zzz
  }
}

Key Points to Remember

  • abstract class = cannot create object directly
  • abstract method = no body, child MUST override it
  • Abstract class CAN have regular methods with body
  • Use override keyword in child class to provide implementation
  • Your Project Example: In Audit Module, you could have an abstract BaseAuditService with abstract method PerformAudit() that each specific audit type implements differently
C# Arrays

Create an Array

Arrays store multiple values in a single variable. Declare with square brackets:

// String array
string[] cars = {"Volvo", "BMW", "Ford", "Mazda"};

// Integer array
int[] myNum = {10, 20, 30, 40};

Access and Change Elements

Array indexes start with 0: [0] = first, [1] = second, etc.

string[] cars = {"Volvo", "BMW", "Ford", "Mazda"};

// Access element
Console.WriteLine(cars[0]); // Output: Volvo

// Change element
cars[0] = "Opel";
Console.WriteLine(cars[0]); // Output: Opel

// Array length
Console.WriteLine(cars.Length); // Output: 4

Loop Through Array

for loop - uses index and Length:

string[] cars = {"Volvo", "BMW", "Ford", "Mazda"};

// for loop
for (int i = 0; i < cars.Length; i++)
{
  Console.WriteLine(cars[i]);
}

foreach loop - easier to write, more readable:

// foreach - no counter needed
foreach (string car in cars)
{
  Console.WriteLine(car);
}

Sort Arrays

// Sort string array (alphabetical)
string[] cars = {"Volvo", "BMW", "Ford", "Mazda"};
Array.Sort(cars);
// Result: BMW, Ford, Mazda, Volvo

// Sort int array (ascending)
int[] myNumbers = {5, 1, 8, 9};
Array.Sort(myNumbers);
// Result: 1, 5, 8, 9

System.Linq - Min, Max, Sum

using System.Linq;

int[] myNumbers = {5, 1, 8, 9};
Console.WriteLine(myNumbers.Max());  // 9 (largest)
Console.WriteLine(myNumbers.Min());  // 1 (smallest)
Console.WriteLine(myNumbers.Sum());  // 23 (total)

Key Points

  • Array index starts from 0
  • Use .Length to get array size
  • for loop when you need index, foreach when you just need values
  • Array.Sort() sorts in place
  • Add using System.Linq; for Min, Max, Sum
  • Arrays are fixed size - use List<T> for dynamic size
C# Break and Continue

Break - Exit the Loop

break stops the loop completely and jumps out.

for (int i = 0; i < 10; i++)
{
  if (i == 4)
  {
    break; // stops loop when i = 4
  }
  Console.WriteLine(i);
}
// Output: 0, 1, 2, 3 (stops at 4, never prints 4-9)

Continue - Skip One Iteration

continue skips the current iteration and moves to the next one.

for (int i = 0; i < 10; i++)
{
  if (i == 4)
  {
    continue; // skips 4, continues with 5
  }
  Console.WriteLine(i);
}
// Output: 0, 1, 2, 3, 5, 6, 7, 8, 9 (4 is skipped)

Break and Continue in While Loop

// Break in while
int i = 0;
while (i < 10)
{
  Console.WriteLine(i);
  i++;
  if (i == 4) { break; }
}
// Output: 0, 1, 2, 3

// Continue in while
int i = 0;
while (i < 10)
{
  if (i == 4)
  {
    i++;
    continue; // skip 4
  }
  Console.WriteLine(i);
  i++;
}
// Output: 0, 1, 2, 3, 5, 6, 7, 8, 9

Key Points

  • break = EXIT the entire loop immediately
  • continue = SKIP current iteration, go to next
  • Both work in for, while, and foreach loops
  • break also used in switch statements
C# Constructors

What is a Constructor?

  • A special method used to initialize objects
  • Called automatically when an object is created with new
  • Must have same name as the class
  • No return type (not even void)
  • Used to set initial values for fields

Default Constructor (No Parameters)

class Car
{
  public string model;

  // Constructor - same name as class, no return type
  public Car()
  {
    model = "Mustang"; // set initial value
  }
}

// Usage
Car Ford = new Car(); // constructor called automatically
Console.WriteLine(Ford.model); // Output: Mustang

Constructor with Parameters

class Car
{
  public string model;
  public string color;
  public int year;

  // Constructor with multiple parameters
  public Car(string modelName, string modelColor, int modelYear)
  {
    model = modelName;
    color = modelColor;
    year = modelYear;
  }
}

// Usage
Car Ford = new Car("Mustang", "Red", 1969);
Console.WriteLine(Ford.color + " " + Ford.year + " " + Ford.model);
// Output: Red 1969 Mustang

Constructors Save Time

Without constructor (more code):

Car Ford = new Car();
Ford.model = "Mustang";
Ford.color = "Red";
Ford.year = 1969;

Car Opel = new Car();
Opel.model = "Astra";
Opel.color = "White";
Opel.year = 2005;

With constructor (clean and short):

Car Ford = new Car("Mustang", "Red", 1969);
Car Opel = new Car("Astra", "White", 2005);

Key Points

  • Constructor name = class name, no return type
  • Called automatically when new is used
  • If you don't write one, C# creates a default empty constructor
  • Can have parameters to set values during creation
  • Can be overloaded (multiple constructors with different parameters)
  • Your Project: In Audit Module, the AuditEntry class constructor takes userId, action, timestamp to initialize each audit record
C# While Loop

While Loop

Executes code as long as condition is True. Checks condition before each iteration.

int i = 0;
while (i < 5)
{
  Console.WriteLine(i);
  i++; // IMPORTANT: increase counter, otherwise infinite loop!
}
// Output: 0, 1, 2, 3, 4

Do/While Loop

Executes code at least once, then checks condition. Checks condition after each iteration.

int i = 0;
do
{
  Console.WriteLine(i);
  i++;
}
while (i < 5);
// Output: 0, 1, 2, 3, 4

// Even if condition is false, runs ONCE:
int x = 10;
do
{
  Console.WriteLine(x); // Output: 10 (runs once!)
}
while (x < 5);

Key Difference

  • while — checks condition FIRST, may never execute if condition is false
  • do/while — executes FIRST, then checks condition. Always runs at least once
  • Always increment your counter (i++) to avoid infinite loops!
C# For Loop

For Loop - Syntax

Use when you know exactly how many times to loop.

for (statement1; statement2; statement3)
{
  // code to execute
}

// statement1 = runs ONCE before loop starts (initialize)
// statement2 = condition to keep running (check)
// statement3 = runs AFTER each iteration (increment)

Examples

// Print 0 to 4
for (int i = 0; i < 5; i++)
{
  Console.WriteLine(i);
}
// Output: 0, 1, 2, 3, 4

// Print only even numbers 0 to 10
for (int i = 0; i <= 10; i = i + 2)
{
  Console.WriteLine(i);
}
// Output: 0, 2, 4, 6, 8, 10

Foreach Loop - For Arrays

Used exclusively to loop through arrays/collections. No counter needed.

string[] cars = {"Volvo", "BMW", "Ford", "Mazda"};

foreach (string car in cars)
{
  Console.WriteLine(car);
}
// Output: Volvo, BMW, Ford, Mazda

Key Points

  • for — use when you know the count (index-based)
  • foreach — use for arrays/collections (simpler, more readable)
  • while — use when you don't know how many times
  • for loop has 3 parts: initialize; condition; increment
  • foreach cannot modify the collection during iteration
C# If ... Else

Conditions and Operators

  • a < b — Less than
  • a <= b — Less than or equal to
  • a > b — Greater than
  • a >= b — Greater than or equal to
  • a == b — Equal to
  • a != b — Not equal to

if Statement

Executes code block if condition is True.

int x = 20;
int y = 18;
if (x > y)
{
  Console.WriteLine("x is greater than y");
}

if ... else

else runs when condition is False.

int time = 20;
if (time < 18)
{
  Console.WriteLine("Good day.");
}
else
{
  Console.WriteLine("Good evening.");
}
// Output: Good evening. (because 20 is NOT less than 18)

if ... else if ... else

Multiple conditions — checks one by one until one is True.

int time = 22;
if (time < 10)
{
  Console.WriteLine("Good morning.");
}
else if (time < 20)
{
  Console.WriteLine("Good day.");
}
else
{
  Console.WriteLine("Good evening.");
}
// Output: Good evening. (22 is not <10, not <20, so else runs)

Ternary Operator (Short-hand if...else)

One-line replacement for simple if/else:

// Syntax: variable = (condition) ? trueValue : falseValue;

int time = 20;
string result = (time < 18) ? "Good day." : "Good evening.";
Console.WriteLine(result); // Output: Good evening.

Key Points

  • if must be lowercase (not If or IF)
  • else if — test new condition when first is false
  • else — runs when ALL conditions above are false
  • Ternary ? : — short-hand for simple if/else in one line
  • Can have multiple else if blocks
C# Inheritance

What is Inheritance?

  • Base Class (parent) — the class being inherited from
  • Derived Class (child) — the class that inherits from another class
  • Use the : symbol to inherit
  • Child gets all fields and methods from parent automatically
  • Why? Code reusability — write once, use in many classes

Example

// Base class (parent)
class Vehicle
{
  public string brand = "Ford";
  public void honk()
  {
    Console.WriteLine("Tuut, tuut!");
  }
}

// Derived class (child) - inherits from Vehicle
class Car : Vehicle
{
  public string modelName = "Mustang";
}

// Usage
Car myCar = new Car();
myCar.honk();  // From Vehicle (parent) - Output: Tuut, tuut!
Console.WriteLine(myCar.brand + " " + myCar.modelName);
// Output: Ford Mustang

Sealed Keyword - Prevent Inheritance

Use sealed to stop other classes from inheriting:

sealed class Vehicle
{
  // ...
}

class Car : Vehicle // ERROR!
{
  // 'Car': cannot derive from sealed type 'Vehicle'
}

Key Points

  • Use : to inherit: class Child : Parent
  • Child gets parent's fields + methods for free
  • C# supports single inheritance only (one parent class)
  • Use sealed to prevent inheritance
  • Your Project: In Audit Module, all controllers inherit from BaseController which has common methods like GetCurrentUser(), LogActivity()
C# Interface

What is an Interface?

  • A completely abstract class — only method signatures, no body
  • Start name with "I" (e.g., IAnimal, IPayment)
  • Members are abstract and public by default
  • Can contain properties and methods, but NOT fields/variables
  • Cannot contain a constructor
  • Cannot create objects from an interface

Implementing an Interface

Use : to implement. Must provide body for ALL methods. No override keyword needed.

// Interface - only signatures
interface IAnimal
{
  void animalSound(); // no body
}

// Class implements the interface
class Pig : IAnimal
{
  public void animalSound()
  {
    Console.WriteLine("The pig says: wee wee");
  }
}

// Usage
Pig myPig = new Pig();
myPig.animalSound(); // Output: The pig says: wee wee

Multiple Interfaces

C# doesn't support multiple class inheritance, but a class CAN implement multiple interfaces:

interface IFirstInterface
{
  void myMethod();
}

interface ISecondInterface
{
  void myOtherMethod();
}

// Implement multiple interfaces with comma
class DemoClass : IFirstInterface, ISecondInterface
{
  public void myMethod()
  {
    Console.WriteLine("Some text..");
  }
  public void myOtherMethod()
  {
    Console.WriteLine("Some other text...");
  }
}

Why Use Interfaces?

  • Security/Abstraction — hide details, show only what's needed
  • Multiple inheritance — class can implement many interfaces
  • Loose coupling — depend on interface, not concrete class
  • Testability — easy to mock interfaces in unit tests

Key Points

  • Interface = contract (what to do, not how)
  • Name starts with I (IAnimal, IService)
  • All methods must be implemented by the class
  • No override keyword needed (unlike abstract class)
  • Supports multiple implementation (comma separated)
  • Your Project: In Preventive Maintenance Module, IVisitService interface defines ScheduleVisit(), GetBranchStatus() — the actual logic is in VisitService class
C# Switch

Switch Statement

Select one of many code blocks to execute based on a value. Cleaner than multiple if...else if.

switch(expression)
{
  case x:
    // code block
    break;
  case y:
    // code block
    break;
  default:
    // runs if no case matches
    break;
}

Example - Weekday

int day = 4;
switch (day)
{
  case 1: Console.WriteLine("Monday"); break;
  case 2: Console.WriteLine("Tuesday"); break;
  case 3: Console.WriteLine("Wednesday"); break;
  case 4: Console.WriteLine("Thursday"); break;
  case 5: Console.WriteLine("Friday"); break;
  case 6: Console.WriteLine("Saturday"); break;
  case 7: Console.WriteLine("Sunday"); break;
}
// Output: Thursday (day = 4)

Default Keyword

Runs when no case matches (optional, like else):

int day = 4;
switch (day)
{
  case 6:
    Console.WriteLine("Today is Saturday.");
    break;
  case 7:
    Console.WriteLine("Today is Sunday.");
    break;
  default:
    Console.WriteLine("Looking forward to the Weekend.");
    break;
}
// Output: Looking forward to the Weekend. (4 doesn't match 6 or 7)

Key Points

  • break — stops execution, exits the switch block
  • default — runs if no case matches (like else)
  • Expression is evaluated once, then compared with each case
  • Without break, code falls through to next case
  • Use switch when you have many fixed values to check (cleaner than if/else if)
  • Your Project: In Helpdesk Tool, switch on ticket type ("Hardware", "Software", "Network") to route to appropriate team
